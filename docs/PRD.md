# 관계부 (Relationship Book) PRD

> 작성자: Product Planner Agent
> 최종 갱신: 2026-05-04
> 버전: v0.1 (M1 MVP 정의 중심)

---

## 1. 한 줄 요약

**관계부(Relationship Book)** 는 흩어진 인맥과 인간관계를 한 곳에서 관리하고, AI가 관계 건강도를 분석해 "지금 누구에게 연락할지"까지 제안해주는 모바일 우선 PWA이다.

---

## 2. 배경 & 문제

### 현재 상황
- 직장인·영업직·프리랜서 등 인맥이 자산인 사람들에게 "누구와 마지막으로 언제 연락했는지" 추적할 도구가 없다.
- 카카오톡/인스타/연락처/캘린더에 정보가 분산되어 있고, 경조사·선물 내역은 메모장이나 머릿속에 있다.
- 기존 CRM은 B2B 영업 관점이라 개인의 친구·가족·지인 관리에 부적합하다.

### 사용자 페인 포인트 (가설)
| # | 페인 | 근거(가설) |
|---|------|-----------|
| P1 | 한참 연락 안 한 사람을 까먹고, 연락 시점을 놓쳐 관계가 흐지부지된다 | 레퍼런스(리얼인맥, 슈퍼프린)가 해결하려는 핵심 문제 |
| P2 | 경조사 챙기다가 누가 누구한테 얼마 줬는지 헷갈린다 | "경조사 다이어리" 앱의 존재 |
| P3 | 받은 선물·취향(MBTI/안 먹는 음식 등)을 기억 못 해 다음에 또 같은 실수를 한다 | 영업/네트워킹 종사자 인터뷰 가설 |
| P4 | 끊어도 될 관계인지, 살려야 할 관계인지 객관적으로 판단할 기준이 없다 | AI 관계 분석에 대한 시장 미충족 수요 |

> [가정] 위 페인은 사전 설문/인터뷰 5명 이상으로 검증 필요. 정량 데이터는 베타 출시 후 보강.

## 3. 타겟 사용자 & 페르소나

### 1차 타겟: 20~50대 직장인/영업직/프리랜서
- 인맥이 곧 성과/소득에 직결됨
- 모바일 우선, 쉬는 시간/이동 중 짧게 사용
- 카톡·인스타 사용 빈도 매우 높음 (한국 시장)

### 페르소나 A — "영업 4년차 김지훈, 32세"
- 고객사 담당자 80명 + 사내 동료 + 친구·가족 = 약 200명
- 분기마다 "한 번이라도 연락해야 하는 사람"을 놓침
- 카톡·캘린더·노션 메모로 분산 관리 → 통합 도구 필요

### 페르소나 B — "프리랜서 디자이너 이수정, 28세"
- 클라이언트 + 동료 디자이너 커뮤니티 = 약 100명
- 경조사 챙기기, 명절 인사, 작업 후속 컨택이 중요
- "관계 건강도"로 우선순위 정해주면 시간 절약

### 페르소나 C — "자영업자 박영호, 45세"
- 단골 손님·거래처·지역 모임 = 약 300명
- 선물 주고받음·경조사 누락이 비즈니스 직격
- 단순한 UI 선호, AI 분석은 부가가치

## 4. 목표 (Goals) & 비목표 (Non-Goals)

### 목표
- **G1.** MVP(M1)에서 사용자가 인물을 등록하고, 연락을 기록하고, 리마인더를 받는 핵심 루프를 7일 내 정착시킨다.
- **G2.** 사용자별 100명 인물 등록 시에도 검색/필터가 1초 이내 동작한다.
- **G3.** AI 관계 건강도 분석으로 "이번 주 연락 추천 인물 Top 5"를 매주 제공해, 연락 행동률 30% 이상을 달성한다.
- **G4.** 모바일 PWA 설치율 D7 기준 40% 이상.

### 비목표 (Non-Goals)
- **NG1.** B2B CRM 기능(영업 파이프라인, 거래 관리)
- **NG2.** 그룹 채팅·메시징 기능 자체 구현 (저장만, 발송은 외부)
- **NG3.** 데스크톱 전용 풀-기능 UI (반응형은 지원하나 모바일이 1차)
- **NG4.** 소셜 네트워크 기능 (인물 간 공개 프로필 공유 등)

## 5. 핵심 가치 제안

> **"연락 잊지 않게, 관계 흐지부지 안 되게, AI가 다음 행동을 제안해주는 나만의 인맥 비서"**

- 기존 메모/엑셀 대비: 자동 리마인더 + AI 분석
- 카톡/인스타 대비: 관계의 맥락(경조사·선물·취향)이 한 화면에
- B2B CRM 대비: 개인 관계 중심, 가볍고 한국 정서(경조사 등) 반영

## 7. Top 5 사용자 시나리오

### 시나리오 1 — "영업맨의 월요일 아침"
지훈은 출근길에 관계부 알림을 받는다. "이번 주 연락 추천: 김부장님(마지막 연락 23일 전), …" Top 5 카드를 보고, 그중 김부장님 카드를 탭해 카톡 보내기로 연결한다. 돌아와 "오늘 통화함" 버튼 한 번으로 연락 로그가 기록되고, 다음 리마인더가 자동 30일 후로 설정된다.

### 시나리오 2 — "결혼식장에서 축의금 기록"
수정은 친구 결혼식장에서 축의금 10만원을 낸다. 관계부에서 친구 카드를 열어 [경조사 추가] → 결혼/축의금 100,000원/2026-05-15 입력. 5초 컷. 나중에 본인이 결혼할 때 "받은 사람" 리스트로 정산.

### 시나리오 3 — "끊어야 할 관계 파악"
영호는 매월 1일 AI 관계 분석 결과를 본다. "관계 건강도 하락 인물: 박○○(6개월 무연락, 일방적 발신만)". 카드를 보고 [관심 낮춤] 또는 [재연결 시도] 액션 중 선택.

### 시나리오 4 — "신규 인물 빠른 등록"
지훈이 술자리에서 명함을 받았다. 관계부 [+] 버튼 → 사진 촬영(OCR 자동 채움) → 이름/회사/관계유형(고객사) → 저장. 3탭 이내 완료.

### 시나리오 5 — "검색으로 빨리 찾기"
수정이 "MBTI ENTP에 디자이너인 친구 누구였더라?" 검색창에 "ENTP 디자이너" 입력 → 필터된 카드 리스트 출력. 1초 이내.

## 9. 범위 (In Scope / Out of Scope)

### In Scope (M1)
- 모바일 우선 PWA (Next.js 15 + Supabase)
- 한국어 UI (영어 i18n 골격은 준비, 콘텐츠는 한국어만)
- 인증, 인물 CRUD, 태그, 연락 로그, 리마인더, 메모, 검색/필터, 기본 AI 분석, 설정, Export
- 멀티 LLM (Claude / Gemini) 추상화 레이어

### Out of Scope (M1, 추후 마일스톤)
- 경조사·선물·메시지 저장 → M2
- 카카오톡 알림톡, 인스타 연동, 캘린더/연락처 동기화 → M3
- 결제/구독 (M1은 무료 베타) → M3+
- 다국어 콘텐츠 (영어/일본어) → 보류
- 데스크톱 전용 화면 → 보류

## 11. 마일스톤

### M1 — MVP (8주)
**목표**: 핵심 루프(인물 등록 → 연락 기록 → 리마인더 → AI 추천) 검증

- 인증 (이메일 + 구글 OAuth)
- 인물 CRUD + 태그 + 메모
- 연락 이력 로그
- 리마인더 생성/조회/완료 + Web Push
- 검색/필터
- AI 관계 건강도 기본 분석 (주 1회 Top 5 추천)
- 설정 (알림, LLM 프로바이더 선택, CSV Export)
- 모바일 PWA 설치 가능
- 베타 사용자 50명 모집

### M2 — 한국형 차별화 (6주)
**목표**: 경조사·선물·메시지로 "한국 정서" 차별화

- 카카오 OAuth 추가
- 경조사 기록 (결혼·장례·생일·기념일)
- 선물 주고받음 + 합계
- 메시지 저장 (텍스트 붙여넣기·이미지 업로드)
- AI 심화 분석 (관계 트렌드, 끊어야 할 관계)
- 휴대폰 연락처 가져오기 (Web Contact Picker API)

### M3 — 외부 연동 (8주)
**목표**: 자동화로 입력 마찰 제거

- 카카오 알림톡 발송 연동
- 구글 캘린더 양방향 동기화
- OS 연락처 동기화 (Android Native Wrapper 검토)
- 인스타그램 조회 연동 (가능 범위 내)
- 무료/유료 플랜 분리 + 결제
