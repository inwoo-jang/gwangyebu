## 4.3 모바일 — 편집 폼 (`/p/[id]/edit`)

추가 폼과 동일하나:
- 헤더: "인물 편집" + 저장 버튼은 변경사항 있을 때만 활성
- 우측 상단 ⋮ 메뉴: "삭제" (destructive)
- 사진 영역에 기존 사진 표시 + "변경" 버튼
- 미저장 변경사항 있을 때 ✕ 탭 시 확인 다이얼로그 "변경사항을 저장하지 않고 나갈까요?"

## 4.4 데스크톱 변형

`Dialog` (max-w-xl) 또는 단독 페이지. 2열 레이아웃 (좌: 사진, 우: 폼).

```
┌──────────────────────────────────────────────────┐
│  새 인물 추가                              ✕     │
├───────────┬──────────────────────────────────────┤
│   👤      │  이름 * [______________________]      │
│   사진     │  관계유형 [친구▾]                    │
│   변경     │  태그 [+]                            │
│           │  생일 [월▾][일▾][연▾]                │
│           │  MBTI [ENFP▾]                        │
│           │  메모 ┌──────────────────────┐        │
│           │       │                      │        │
│           │       └──────────────────────┘        │
│           │  알림주기 ( )없음 (●)1개월 ( )3개월   │
└───────────┴──────────────────────────────────────┘
                                    [취소] [저장]
```

## 4.5 인터랙션 / 액션

| 요소 | 동작 |
|------|------|
| 사진 + 버튼 | OS 권한 요청 → 카메라/갤러리 선택 (Web File API) |
| 사진 업로드 | 5MB 이하 검증, EXIF 제거, Supabase Storage 업로드 |
| 사진 변경 (편집) | 기존 사진 prevew + [변경] [제거] 버튼 |
| 이름 입력 | 실시간 1~50자 카운터, 동명이인 있을 시 blur 후 안내 배너 |
| 관계유형 | 토글 칩 단일 선택, 미선택 시 'etc' |
| 태그 | 입력 + 엔터로 추가, 자동완성 드롭다운, 최대 10개 |
| 생일 | Select 컴포넌트 — 2월 30일 등 자동 검증 |
| MBTI | Select (16종 + 모름) |
| 알림 주기 | RadioGroup, 직접 입력 시 inline number input (1~365) |
| 저장 | Optimistic UI: 즉시 카드 표시 + 백그라운드 저장 |
| 저장 실패 | 에러 토스트 + 폼 보존 + 재시도 |
| ✕ 닫기 (변경사항 있음) | AlertDialog "변경사항을 저장하지 않고 나갈까요?" |
| 편집 메뉴 ⋮ → 삭제 | AlertDialog 확인 → soft-delete + 토스트 "삭제했어요. [실행 취소]" |

## 4.6 사용 shadcn 컴포넌트

| 영역 | 컴포넌트 |
|------|----------|
| Sheet 추가 방식 | `Sheet` side="bottom" |
| 폼 전체 | `Form` (react-hook-form 통합) |
| 입력 필드 | `Input`, `Textarea`(추가 설치), `Select` |
| 라디오 그룹 | `RadioGroup`(추가 설치) |
| 체크박스/스위치 | `Switch`(추가 설치) |
| 사진 영역 | `Avatar` + 커스텀 업로드 |
| 토글 칩 | `Badge` 또는 `ToggleGroup`(추가 설치) |
| 태그 입력 | 커스텀 (Combobox 패턴) |
| 변경 확인 | `AlertDialog`(추가 설치) |

## 4.7 데이터 의존성

| 필드 | 엔티티 | 비고 |
|------|--------|------|
| 모든 입력 | `Person` 행 | feature-spec.md F-PERSON 참조 |
| 태그 | `Tag` + `PersonTag` | 자동완성: 사용자 보유 태그 + 최근 사용 |
| 사진 | `Storage` URL → `Person.photo_url` | EXIF 제거 후 업로드 |
| 알림 주기 | `Person.reminder_interval_days` | 0=없음 |

자동완성 데이터:
- 태그 자동완성: `Tag where user_id=auth.uid() order by usage_count desc limit 10`
- 동명이인 검사: `Person where name={name} and user_id=auth.uid() and deleted_at is null`

## 4.8 검증 룰

| 필드 | 검증 |
|------|------|
| 이름 | 필수, 1~50자, 공백만 차단 |
| 사진 | 5MB, jpg/png/webp |
| 태그명 | 1~20자, 공백/이모지만 차단, 사용자별 unique |
| 태그 개수 | 최대 10개 |
| 생일 | 월(1~12), 일(1~31, 월별 검증), 연도(1900~현재) |
| 메모 | 0~2000자, 카운터 표시 |
| 알림주기 (직접) | 1~365 정수 |

## 4.9 상태별 처리

| 상태 | 화면 |
|------|------|
| 입력 중 | 자동저장 안 함 (M1), draft는 IndexedDB에 임시 보관 |
| 사진 업로드 중 | Avatar 자리에 progress overlay |
| 저장 중 | 저장 버튼 spinner + 비활성 |
| 오프라인 | 토스트 "오프라인이에요. 연결되면 동기화돼요" + IndexedDB 큐 |
| 동명이인 발견 | 저장 시 AlertDialog "동명이인 OOO이 이미 있어요. 그래도 추가할까요?" |
| 사진 5MB 초과 | 인라인 에러 "5MB 이하만 업로드 가능해요. 압축할까요?" + [압축] |
| 폼 검증 실패 | 첫 에러 필드로 스크롤 + 인라인 에러 + 토스트 "확인이 필요한 항목이 있어요" |
