## 6.7 인터랙션 / 액션

| 요소 | 동작 |
|------|------|
| 검색 입력 | 디바운스 200ms, 1자 이상부터 쿼리 |
| 초성 검색 | "ㅎㄱㄷ" → "홍길동" 매칭 (별도 인덱스) |
| ✕ (검색 클리어) | 입력 비움 + 6.1 화면 |
| 최근 검색어 탭 | 해당 검색어로 재검색 |
| 최근 검색어 ✕ | 개별 삭제 (localStorage) |
| 필터 칩 탭 | Sheet (모바일) / Popover (데스크톱) |
| 자주 쓰는 필터 | 원탭에 필터 적용 |
| 정렬 ▾ | 관련도 / 가나다 / 최근 연락 / 추가일 (Sheet) |
| 결과 카드 탭 | `/p/{id}` |
| 결과 카드 길게 누름 | 컨텍스트 메뉴 (방금 연락함, 리마인더 등) |
| 0건 결과 [+ 새 인물 추가] | `/p/new?name={query}` |
| 필터 적용 | URL 쿼리에 반영 (공유/북마크 가능) |

## 6.9 데이터 의존성

### 검색 인덱싱 (data-model.md 참조)
- Postgres `tsvector` (name, memo, mbti) — 한국어 분석기 또는 ngram
- 초성 인덱스: name_chosung 컬럼 (별도) 또는 trigram extension
- 사용자별 RLS

### 쿼리 패턴
```sql
-- 텍스트 검색
SELECT * FROM person
WHERE user_id = auth.uid()
  AND deleted_at IS NULL
  AND (
    name ILIKE '%' || $query || '%'
    OR name_chosung LIKE $chosung_query
    OR mbti = $upper_query
    OR EXISTS (SELECT 1 FROM tag JOIN person_tag ... WHERE tag.name ILIKE $query)
  )
ORDER BY ts_rank(...), last_contact_at DESC
LIMIT 50;
```

### 인덱스 필수
- `idx_person_user_name_trgm` (gin)
- `idx_person_user_chosung_trgm` (gin)
- `idx_person_user_last_contact` (btree desc)

## 6.10 상태별 처리

| 상태 | 화면 |
|------|------|
| 검색어 없음 | 6.1 (최근/추천 필터) |
| 1자 미만 | 검색 미실행, 최근만 표시 |
| 입력 중 (디바운스) | 결과 영역에 미니 spinner |
| 결과 0건 | 6.5 빈 상태 + 추가 CTA |
| 필터 조합 0건 | "모든 필터를 풀면 결과가 나와요" + [필터 풀기] |
| 500명+ 결과 | 가상 스크롤 + 페이지당 50건, "더 보기" |
| 오프라인 | IndexedDB 캐시 검색 (이름만), 상단 배너 "오프라인 — 일부 결과만 표시" |
| 특수문자만 입력 | "검색어를 입력해주세요" 안내 |
