## 7.8 인터랙션 / 액션

| 요소 | 동작 |
|------|------|
| ⋮ 메뉴 | 다시 분석 / AI 프로바이더 변경 / 분석 빈도 설정 |
| [🔄 지금 다시 분석] | 즉시 분석 실행 (주 3회 한도) — 한도 초과 시 다음 가능 시각 안내 |
| 추천 카드 헤더 (이름) | `/p/{id}?from=ai` 인물 상세 |
| [💬 지금 연락] | 7.2 채널 선택 Sheet → 외부 앱 → 복귀 시 ContactLog 모달 자동 |
| [⏰ 리마인더] | 리마인더 생성 모달 (인물 prefill) |
| [👎 이 추천 별로에요] | AIFeedback 저장 → 다음 분석에서 가중치 하향 + 카드 페이드아웃 |
| 점수 / 사유 영역 탭 | 7.3 Dialog (점수 산출 근거) |
| 인사이트 [보기] | 점수 하락 인물 필터 검색 (`/search?filter=score_drop`) |
| 푸시 알림 탭 | `/ai/recommendations`로 직진 |
| Pull-to-refresh | 캐시 갱신 (실제 분석은 다시 실행 안 함) |

## 7.10 데이터 의존성

| 영역 | 엔티티 |
|------|--------|
| 추천 리스트 | `Recommendation where user_id=auth.uid() and week=current order by rank limit 5` |
| 인물 정보 | `Person where id in (rec.person_id)` 조인 |
| 점수 | `RelationshipScore where person_id=rec.person_id` |
| 사유 | `Recommendation.reason` (LLM 생성, 한국어 1~2문장) |
| 마지막 연락 | `Person.last_contact_at` |
| 사용자 피드백 | `AIFeedback where user_id=auth.uid()` (가중치 영향) |
| 토큰 사용량 | `ProviderUsage where user_id=auth.uid() and month=current` |

생성/갱신 트리거:
- 매주 월요일 09:00 KST (Edge Function cron)
- 사용자 수동 트리거 (한도 주 3회)
- 인물 5명 미만 / 연락 로그 10건 미만이면 미생성

## 7.11 상태별 처리

| 상태 | 화면 |
|------|------|
| 인물 5명 미만 | 7.4 분석 부족 |
| 연락 로그 10건 미만 | "연락 기록이 더 쌓이면 추천을 시작할게요" |
| 분석 미실행 (월요일 전) | "월요일 09시에 첫 추천이 도착해요" + 미리보기 |

| 추천 톤이 어색 | 7.3 Dialog [이 분석 잘못됐어요] → AIFeedback type='wrong' |

## 7.12 한국어 카피

| 위치 | 카피 |
|------|------|
| 헤더 | "이번 주 추천" |
| 분석 메타 | "✨ {날짜} 분석 · {프로바이더} · 다음 {다음날짜}" |
| 분석 중 | "분석 중이에요 (~30초)" |
| 빈 상태 | "조금만 더 등록하면 추천을 시작할게요" |
| 푸시 | "이번 주 연락 추천 5명이 도착했어요" |
| 추천 사유 톤 | 부드럽고 강요하지 않게 — "어때요?", "한 번 안부 전해볼까요?" |
| 부정 피드백 | "이 추천이 마음에 안 들어요" → "알려주셔서 감사해요. 다음엔 더 나아질게요" |
| Quota | "이번 달 한도에 도달했어요. 다음 갱신은 6/1이에요." |
| 외부 앱 안내 | "외부 앱에서 돌아오면 연락 기록을 자동으로 제안해요" |
