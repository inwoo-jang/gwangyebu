## 1.1 랜딩 (비로그인)

### 모바일 (360x640)

```
┌─────────────────────────────┐
│                             │
│                             │
│        🤝                   │  ← 일러스트 (서로 손 잡은 두 사람)
│                             │
│   잊지 않게,                │  ← 헤드라인 (text-3xl, font-bold)
│   흐지부지되지 않게.        │
│                             │
│   인맥과 마음을 한 곳에.    │  ← 서브 (text-base, muted)
│                             │
│                             │
├─────────────────────────────┤
│  [  지금 시작하기  ]        │  ← Primary CTA (h-12, w-full)
│                             │
│  이미 계정이 있어요  →      │  ← 텍스트 링크
│                             │
│  ──── 또는 ────             │
│                             │
│  [ 🟢 구글로 계속하기 ]     │  ← Outline button
│  [ 🟡 카카오로 계속하기 ]   │  ← (M2 — M1에는 비활성)
│                             │
│  가입 시 [이용약관]·[개인정보] │
│  에 동의하게 됩니다.        │
│                             │
└─────────────────────────────┘
```

### 데스크톱

좌(헤드라인+CTA) / 우(스크린샷 mockup) 2단 레이아웃, max-w-5xl, 중앙 정렬.

### 액션
| 요소 | 동작 |
|------|------|
| 지금 시작하기 | `/auth/sign-up` 이동 |
| 이미 계정 있어요 | `/auth/sign-in` 이동 |
| 구글로 계속 | Supabase OAuth `provider: google` |
| 카카오로 계속 (M2) | Supabase OAuth `provider: kakao` (M1: 비활성, "준비 중" 툴팁) |
| 약관/개인정보 링크 | `/legal/terms`, `/legal/privacy` 이동 |

### 사용 shadcn 컴포넌트
`Button`, `Separator`(필요 시 div + border-t)

## 1.2 회원가입 (이메일)

### 모바일

```
┌─────────────────────────────┐
│  ←  회원가입                │  ← Header (back)
├─────────────────────────────┤
│                             │
│  이메일                     │
│  ┌───────────────────────┐  │
│  │ you@example.com       │  │  ← Input
│  └───────────────────────┘  │
│                             │
│  비밀번호                   │
│  ┌───────────────────────┐  │
│  │ ••••••••••       👁   │  │  ← Input + show/hide
│  └───────────────────────┘  │
│  ●●●○○ 강함                 │  ← Strength bar
│  · 8자 이상 · 영문+숫자     │  ← 헬퍼 텍스트
│                             │
│  표시 이름 (선택)           │
│  ┌───────────────────────┐  │
│  │ 인우                  │  │
│  └───────────────────────┘  │
│                             │
│  ☐ [이용약관]·[개인정보]에   │  ← Checkbox + 링크
│    동의합니다 (필수)        │
│                             │
│  ☐ 마케팅 정보 수신 (선택)  │
│                             │
├─────────────────────────────┤
│  [    가입하기    ]         │  ← Primary, 비활성: 약관 미체크
└─────────────────────────────┘
```

### 액션
| 요소 | 동작 |
|------|------|
| 이메일 입력 | RFC 5322 검증, blur 시 인라인 에러 |
| 비밀번호 | 8~64자, 영문+숫자 검증, 강도 표시바 (zxcvbn) |
| 표시 이름 | 선택, 미입력 시 이메일 prefix 사용 |
| 약관 체크 | 필수, 미체크 시 가입 버튼 비활성 |
| 가입하기 | Supabase signUp 호출 → 인증 메일 안내 화면 |

### 에러/엣지
- 이미 가입된 이메일 → Dialog "이미 가입된 계정이에요. 로그인하시겠어요?" + [로그인하기]
- 5회 실패 → 5분 잠금 안내
- 약한 비밀번호 → 강도 ●●○○○ 빨간색 + "더 강력한 비밀번호를 사용해주세요"

### 사용 shadcn 컴포넌트
`Form`, `Input`, `Label`, `Button`, `Checkbox` (추가 설치 필요), `Dialog`

```
┌─────────────────────────────┐

│                             │

## 1.4 온보딩 1단계 — 사용 목적

### 모바일

```
┌─────────────────────────────┐
│  ●○○                  건너뛰기 │  ← 진행도 + skip
├─────────────────────────────┤
│                             │
│  관계부를                   │  ← H1
│  어떻게 쓰실 건가요?        │
│                             │
│  목적에 맞춰 추천 톤을 맞춰 │  ← muted
│  드릴게요. (복수 선택 OK)   │
│                             │
│  ┌─────────────────────┐    │
│  │ 💼 영업 / 네트워킹   │    │  ← Chip (tap to toggle)
│  └─────────────────────┘    │
│  ┌─────────────────────┐    │
│  │ 🤍 친구·가족 챙김    │    │  ← selected: bg-primary/10
│  └─────────────────────┘    │
│  ┌─────────────────────┐    │
│  │ 🎁 경조사 정리       │    │
│  └─────────────────────┘    │
│  ┌─────────────────────┐    │
│  │ ✨ AI 추천 받아보기  │    │
│  └─────────────────────┘    │
│                             │
├─────────────────────────────┤
│  [    다음    ]             │  ← 비활성: 0개 선택

### 데이터 의존성
- 저장 위치: `User.onboarding.purposes: string[]`
- 사용처: AI 분석 톤 가중치 초기값

## 1.6 온보딩 3단계 — 첫 인물 추가 유도

```
┌─────────────────────────────┐
│  ●●●                  건너뛰기 │
├─────────────────────────────┤
│                             │
│       👤                    │  ← 일러스트
│                             │
│  첫 사람을                  │
│  등록해볼까요?              │
│                             │
│  이름만 있어도 충분해요.    │
│  나머지는 천천히 채워도 OK. │
│                             │
│                             │
├─────────────────────────────┤
│  [  지금 한 명 추가하기  ]  │  ← Primary → /p/new
│  [   나중에 할게요    ]     │  ← Ghost → /home
└─────────────────────────────┘
```

### 액션
| 요소 | 동작 |
|------|------|
| 지금 추가 | `/p/new?from=onboarding` |
| 나중에 | `/home` (빈 상태 안내 표시) |
| 건너뛰기 | 1·2단계 결과 저장 후 `/home` |

## 1.7 상태별 처리

| 상태 | 처리 |
|------|------|
| 로딩 (OAuth 콜백 중) | 풀스크린 spinner + "잠시만요..." |
| OAuth 거부/취소 | 토스트 "로그인이 취소되었어요" + 랜딩 복귀 |
| 약관 미동의 | CTA 비활성, 인라인 에러 "약관에 동의해주세요" |
| 온보딩 도중 이탈 | 다음 진입 시 마지막 단계부터 재개 (User.onboarding.last_step) |
| 5명 미만 인물 등록 후 AI 화면 진입 | "조금만 더 등록하면 추천을 시작할게요" |

---

## 1.8 사용 shadcn 컴포넌트 매핑

| 화면 영역 | 컴포넌트 |
|-----------|----------|
| 모든 버튼 | `Button` (variant: default/outline/ghost) |
| 폼 입력 | `Form`, `Input`, `Label` |
| 약관 체크박스 | `Checkbox` (추가 설치) |
| 진행 다이얼로그 | `Dialog` |
| 칩(목적/태그) | `Badge` 또는 커스텀 (variant: outline / selected) |
| 헤더 back | `Button` variant=ghost size=icon |
