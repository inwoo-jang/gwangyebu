# PRD 작성 스킬

## 표준 PRD 구조

```markdown
# {제품명} PRD

## 1. 한 줄 요약
## 2. 배경 & 문제
- 현재 상황
- 사용자 페인 포인트 (근거: 인터뷰/설문/데이터)
## 3. 타겟 사용자 & 페르소나
## 4. 목표 (Goals) & 비목표 (Non-Goals)
## 5. 핵심 가치 제안
## 6. 핵심 기능 (P0/P1/P2)
| ID | 기능 | 우선순위 | 근거 |
## 7. 사용자 시나리오 (Top 3~5)
## 8. 성공 지표
- 북극성: ...
- 보조 지표: ...
## 9. 범위 (In Scope / Out of Scope)
## 10. 의존성 & 리스크
## 11. 마일스톤
- M1 (MVP): ...
- M2: ...
## 12. 열린 질문 / 가정
```

## 작성 원칙

- 모호한 형용사("쉬운", "빠른") 대신 측정 가능한 기준으로 표현
- 기능은 **사용자 가치**로 시작, 그 다음 **솔루션** 서술
- 한국어. 영문 용어는 첫 등장 시 괄호 병기
- P0는 MVP에 반드시 들어가야 하는 것만
