# 유저플로우 작성 스킬

## 산출물 형식

각 핵심 시나리오마다:

```markdown
## 시나리오: {이름}
**전제**: 로그인 상태, ...
**목표**: ...

### 플로우
\`\`\`mermaid
flowchart TD
    A[홈] --> B{인물 선택?}
    B -->|예| C[인물 상세]
    B -->|아니오| D[검색]
\`\`\`

### 단계 상세
1. **홈** — 사용자가 보는 것: ..., 가능한 액션: ...
2. **인물 상세** — ...

### 엣지 케이스
- 데이터 없음 / 네트워크 오류 / 권한 거부
```

## 다뤄야 하는 시나리오 (관계부 기준)

1. 신규 가입 + 온보딩 (관계 라벨 초기화)
2. 인물 추가 (수동/연락처 가져오기)
3. 연락 기록 + 리마인더 설정
4. 경조사/선물 기록
5. AI 관계 분석 결과 확인 + 액션 추천 수락
6. 검색/필터 (태그, MBTI, 마지막 연락일)
7. 알림 수신 → 앱 진입

## 원칙

- 사용자 동사로 표현 ("인물을 본다" X → "인물 카드를 탭한다" O)
- 의사결정 분기는 다이아몬드(Mermaid `{}`)로
- 각 화면 → 다음 화면의 트리거(버튼, 자동 이동) 명시
